HTTP - це стандартний протокол передачі даних в Інтернеті, який використовується для передачі гіпертекстових документів, 
таких як HTML, між веб-сервером і клієнтським браузером. Основні методи HTTP:

GET - використовується для отримання даних з сервера; GET-запити не змінюють стан сервера або даних.
POST - використовується для надсилання даних на сервер для обробки; POST-запити зазвичай використовуються для створення нових записів або внесення змін на сервері.
PUT - використовується для оновлення даних на сервері; PUT-запити зазвичай використовуються для зміни існуючих записів.
DELETE - використовується для видалення даних на сервері; запит DELETE використовується для видалення існуючого запису.
Взаємодія клієнта і сервера
У контексті веб-розробки клієнт - це програмне забезпечення, яке взаємодіє з користувачем і надсилає HTTP-запити на сервер для отримання відповіді. Сервер - це програмне забезпечення, яке отримує та обробляє HTTP-запити від клієнта та надсилає відповідь.

Клієнт і сервер спілкуються за допомогою HTTP-запитів і відповідей. Клієнт надсилає HTTP-запит на сервер, 
який містить різні методи і дані, які він бажає передати. Сервер обробляє ці запити і надсилає відповідь, 
яка може містити різні дані і статуси для інформування клієнта про результат виконання запиту.
